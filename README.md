🧑‍🏫 TA Notes
The main thing students need to get out of this activity is how to take a messy, real-world feature request and turn it into a clean system design — specifically identifying the right classes, their attributes, and how they relate to each other before writing a single line of code. The spot where students struggle the most is the Order class, either forgetting to handle an empty order (which causes a crash or returns None instead of 0) or accidentally summing item names instead of prices. Copilot was genuinely useful for generating the initial class scaffolds quickly and helping draft pytest fixtures, which saved a lot of setup time. Where it got misleading was in the UML diagram — it kept adding extra classes like Cart or RecommendationEngine that weren't in the spec at all, so students who just accepted the output without reading it ended up building things they didn't need. If a student comes to me stuck on why their filter method isn't working, I'd ask them to tell me what the method is supposed to return and then have them add a print statement right before the return — nine times out of ten they either forgot the return statement or it's returning the wrong thing, and they find it themselves.
